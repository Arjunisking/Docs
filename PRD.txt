Document 1: PRD (Product
Requirement Document)
What is a PRD?
A Product Requirement Document (PRD) is a comprehensive blueprint that defines what your
website will do, who it's for, and what problems it solves. It's the foundation of your entire
project.
Why You Need a PRD:
• Provides clear direction to AI builders
• Ensures all features are purposeful
• Prevents scope creep and confusion
• Acts as a reference throughout development
What to Include in Your PRD
1. Project Overview
• Project name and brief description
• Business goals and objectives
• Target audience and user personas
2. Core Features
• List of all website features (e.g., contact form, booking system, payment integration)
• Priority level for each feature (Must-have, Nice-to-have)
• Detailed description of how each feature should work
3. Pages Structure
• Number of pages
• Name and purpose of each page
• Navigation flow between pages
4. User Journey
• Step-by-step flow of how users will interact with the site
• Key actions users should take (e.g., sign up, purchase, contact)
5. Success Metrics
• How you'll measure if the website is successful
• Key performance indicators (KPIs)
Example: PRD for a Cafe Website
Project Name: "The Coffee Corner" Website
Goal: Create an engaging online presence to attract customers, showcase menu,
enable online reservations, and increase foot traffic to the physical cafe.
Target Audience:
• Coffee lovers aged 18-45
• Remote workers looking for workspace
• Local residents and office workers
Pages (5 total):
1. Home - Hero section, featured items, atmosphere photos
2. Menu - Complete food and beverage menu with prices and descriptions
3. About - Cafe story, values, team introduction
4. Reservations - Table booking system with date/time selection
5. Contact - Location map, hours, contact form
Core Features:
• Online table reservation system (Must-have)
• Menu with filtering (coffee, food, desserts) (Must-have)
• Gallery showcasing cafe ambiance (Must-have)
• Newsletter signup (Nice-to-have)
• Customer reviews section (Nice-to-have)
Success Metrics:
• 50+ reservations per week through website
• Average session duration: 2+ minutes
• 30% conversion rate from visitor to reservation
AI Prompts to Create Your PRD
Use these prompts with ChatGPT, Claude, or any AI assistant to generate your PRD:
Prompt 1: Initial PRD Creation
"I want to build a website for [YOUR BUSINESS/PROJECT TYPE]. Create a
comprehensive Product Requirement Document (PRD) that includes:
1. Project overview and goals
2. Target audience definition
3. Complete list of required pages
4. Core features with priority levels
5. User journey mapping
6. Success metrics
The business is [DESCRIBE YOUR BUSINESS - e.g., a specialty coffee shop in
downtown area that also serves as a coworking space].
Format the PRD in a clear, structured way that I can share with developers or AI
building tools."
Prompt 2: Refine Specific Sections
"Based on the PRD you created, expand the [SECTION NAME - e.g., Core Features]
section. For each feature, include:
• Detailed description of functionality
• User interaction flow
• Any technical requirements
• Priority level (Must-have, Should-have, Nice-to-have)
• Success criteria for this feature"
Prompt 3: Add Competitive Analysis
"Add a competitive analysis section to the PRD. Research and compare with 3 similar
websites in the [INDUSTRY] industry. For each competitor, identify:
• What they do well that we should consider
• What they're missing that we can capitalize on
• Unique features we can offer to differentiate ourselves
