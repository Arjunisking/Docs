Document 2: Design Document
What is a Design Document?
A Design Document specifies the visual and user experience aspects of your website. It
ensures the AI builder understands exactly how your website should look, feel, and behave.
Why You Need a Design Document:
• Creates visual consistency across all pages
• Defines brand identity and aesthetics
• Guides AI in making design decisions
• Ensures professional and polished output
• Prevents generic, template-like designs
What to Include in Your Design Document
1. Visual Style & Mood
• Overall aesthetic (modern, minimalist, playful, professional, etc.)
• Mood and emotion you want to convey
• Design inspiration references
2. Color Palette
• Primary colors (with hex codes)
• Secondary colors
• Accent colors
• Background colors
• Text colors
3. Typography
• Heading fonts (H1, H2, H3)
• Body text font
• Font sizes and weights
• Line spacing
4. Layout & Structure
• Grid system (if any)
• Spacing and padding standards
• Responsive breakpoints
• Component placement preferences
5. UI Components
• Button styles
• Form elements
• Cards and containers
• Navigation menu design
• Icons and imagery style
6. Page-Specific Designs
• Hero section design
• Layout for each page type
• Special sections or features
7. Reference Images
• Screenshots from Dribbble, Behance, or similar sites
• Competitor websites you like
• Color scheme examples
Example: Design Document for Cafe Website
Visual Style: Modern, warm, and inviting with earthy tones. Clean and minimalist
to highlight coffee and food imagery.
Color Palette:
• Primary: #8B4513 (Rich Brown - coffee color)
• Secondary: #F5E6D3 (Cream - warm background)
• Accent: #D4A373 (Light Coffee)
• Text: #2C1810 (Dark Brown)
• Background: #FFFFFF (White)
Typography:
• Headings: 'Playfair Display' - Elegant serif font
• Body: 'Inter' - Clean sans-serif
• H1: 48px bold
• H2: 32px semi-bold
• Body: 16px regular
Layout Style:
• Maximum width: 1200px centered
• Grid: 12-column responsive
• Generous white space
• Large, high-quality images
Components:
• Buttons: Rounded (8px radius), brown background, cream text, subtle shadow
• Cards: White background, thin border, soft shadow, 12px radius
• Navigation: Fixed top, transparent with blur, turns solid on scroll
Hero Section:
• Full-width background image of cafe interior
• Centered text overlay with gradient
• Large heading + subtitle + CTA button
• Subtle scroll indicator
Design References:
• Dribbble: "Coffee Shop Landing Page" by [Designer Name]
• Reference similar to: bluebottlecoffee.com for clean aesthetic
• Menu layout inspired by: sweettooth.cafe
AI Prompts to Create Your Design Document
Method 1: Using Reference Images (Recommended)
Find inspiration on Dribbble, Behance, or existing websites. Take screenshots and use this
prompt:
Prompt 1: Image-Based Design Document
[Upload 2-3 reference images]
"Based on these design references, create a comprehensive Design Document for
my [PROJECT TYPE] website. Extract and document:
1. Color palette (provide exact hex codes from the images)
2. Typography style and recommended fonts
3. Layout structure and grid system
4. Component styles (buttons, cards, forms)
5. Overall visual mood and aesthetic
6. Spacing and sizing guidelines
Make it detailed enough that a developer or AI tool can recreate this exact visual
style."
Prompt 2: Text-Based Design Document
"Create a detailed Design Document for a [PROJECT TYPE] website with the
following characteristics:
• Visual style: [modern/minimalist/bold/elegant/playful]
• Mood: [professional/friendly/luxurious/energetic]
• Industry: [your industry]
• Target audience: [your audience]
Include:
1. Complete color palette with hex codes
2. Typography recommendations (fonts and sizes)
3. Layout structure and spacing
4. UI component specifications
5. Responsive design considerations
6. Page-specific design guidelines"
Prompt 3: Component-Specific Design
"For the Design Document, create detailed specifications for these specific
components:
1. Navigation Bar
2. Hero Section
3. Call-to-Action Buttons
4. Cards/Content Blocks
5. Forms
6. Footer
For each component, specify: dimensions, colors, typography, hover states, and
responsive behavior.
