# AI Website Planning Docs

A structured document kit for planning and building websites with AI tools.

This repository contains the three core documents needed to guide AI website builders properly:

- PRD (Product Requirement Document)
- Design Document
- Tech Stack Document

It also includes a simple usage guide showing how to use these documents with tools like ChatGPT, Claude, Bolt, Cursor, v0, and similar builders.

## Why this repo exists

Most AI-built websites fail for one simple reason: the input is vague.

This repo solves that by giving AI clear project context across three layers:

- What the website should do
- How the website should look and feel
- What technologies should be used to build it

Together, these documents help produce cleaner, more consistent, and more production-ready website outputs.

1. PRD

The Product Requirement Document defines the project foundation.

It includes:

Project overview
Business goals
Target audience
Page structure
Core features

2. Design Document

The Design Document defines the visual and UX direction of the website.

It includes:

Visual style and mood
Color palette
Typography
Layout structure
UI component guidance
Page-specific design ideas
Reference inspiration

Use this document to tell the AI how the website should look and behave.

3. Tech Stack Document

The Tech Stack Document defines the technologies and tools that should be used.

It includes:

Frontend framework
Meta-framework
Styling solution
Backend options
Libraries and packages
Deployment setup
Additional tools for SEO, analytics, and performance

Use this document to tell the AI what the website should be built with.
